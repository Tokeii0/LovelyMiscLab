# 2026-07-02

## 更新 README（LovelyMiscLab）
- 项目：面向 CTF Misc 的节点式可视化工作台，对标 ComfyUI 的搞法（数据驱动、可视化编排、节点可扩展，把"生成图像"换成"Misc 分析"）。**不要提 CyberChef。**
- 架构：Cargo workspace，`crates/misclab-core`（纯 Rust 引擎，Tauri 无关，可无头单测）+ `src-tauri`（薄适配层）+ `src/`（React 19 前端）。
- NodeDescriptor 是数据驱动 UI 的单一真相源；100+ 内置节点；GraphExecutor 拓扑排序+内容寻址增量缓存；支持 AI 生成流程、复合模块、脚本节点。
- 前端栈：React19/Vite7/Tailwind v4/@xyflow/react/zustand/TanStack Query/React Router。
- 顶部美化：居中 logo `<img src="src-tauri/icons/Square310x310Logo.png" width=160>` + 🧪 标题 + slogan + 6 个 shields.io 徽章。
- 命令：`cargo test -p misclab-core`（测试）；`pnpm tauri dev`（开发）；`pnpm dev`（浏览器预览）；`pnpm tauri build`（构建）。

## 发布配置改为单文件绿色版
- 决策：Tauri "single file" = 独立 .exe，依赖系统 WebView2，无需安装包。
- `tauri.conf.json` 里 `bundle.targets` 设为 `[]`（不产安装包）。二进制名：`misclab-app.exe`（`target/release/`）。
- 重写 `.github/workflows/release.yml`：弃用 tauri-action（面向安装包），改为 `pnpm tauri build` → exe 重命名为 `LovelyMiscLab-<version>-windows-x64.exe` → `softprops/action-gh-release@v2` 发布 draft release 并上传 exe。ci.yml 不变。
